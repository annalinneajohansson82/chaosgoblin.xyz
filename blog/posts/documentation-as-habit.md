ADHD working memory is unreliable. My solution: document everything immediately. Not for others—for future me. The best time to document code is when you understand it. The worst time is when you need to understand it again and can't remember why you made that choice.

<!--more-->

I write docstrings obsessively. I leave comments explaining the why, not the what. I maintain a decision log for major technical choices. This looks like excellent engineering practice, but it's actually accommodation.

The irony: this habit makes my code more maintainable for everyone. What started as a workaround for my own brain became a team standard.
