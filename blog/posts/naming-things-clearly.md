title: Naming things clearly saves cognitive load
date: 2024-09-18
tags:

There are only two hard things in Computer Science: cache invalidation and naming things. For me, there's a third: remembering what my own abbreviations mean. So I stopped using them.

<!--more-->

`getUserById` is clearer than `getUBI`. `temporaryBuffer` is better than `tmpBuf`. The extra characters cost nothing. The cognitive load of decoding abbreviations costs me real time and focus.

This is another habit that looks like good engineering practice but is actually disability accommodation. Explicit, verbose naming isn't just better for teams—it's necessary for my own ability to maintain the code I write.

The best code is the code you can understand six months later without context.
